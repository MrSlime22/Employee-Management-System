Decide Framework

D - Determine the goals


• Assess the effectiveness and efficiency of the Employee Management System.


• Evaluate user satisfaction and overall usability


• Measure the system’s impact on organizational productivity.



E - Explore the questions


• How well does the system meet user needs for managing employee records?


• Are the functionalities (CRUD, search, export) intuitive and accessible?


• What are the common issues or challenges faced by users?


• How does the system perform in terms of speed and reliability?



C - Choose the evaluation approach and methods


In our evaluation approach, we will conduct a collaborative assessment of our system. Each group member will take turns using the prototype, providing feedback on its usability and functionality.


We will hold a structured discussion after each member has interacted with the application. During this discussion, we will share observations, insights, and suggestions for improvement based on our experiences with the prototype.


This hands-on evaluation will help us collectively assess the strengths and weaknesses of our system, allowing us to make informed adjustments for future iterations.



I - Identify the practical issues


• We ensure adequate time for conducting thorough evaluations.


• Encourage participation from users to gather diverse feedback.


• Address any concerns about privacy during the evaluation process.

E - Evaluation, Analyze, Interpret and Present the data


In this evaluation, we use the Nielsen Norman Group Heuristic Evaluation Workbook to identify usability issues within the Employee Management System. By applying Jakob Nielsen’s 10 Usability Heuristics, we assess how well the interface meets user needs and where improvements can be made. Below, we outline the key issues found during the evaluation, along with recommendations for enhancing the overall user experience.


1. Visibility of System Status

Issues: The system doesn’t clearly show what’s happening after user actions. For example, when a user adds or deletes an entry, there’s no feedback to indicate if the action was successful or still in progress.

Recommendations: To fix this, we should add loading indicators and feedback messages. This way, users will know that their action is being processed, which helps them feel more in control.


2. Match Between System and the Real World

Issues: Some of the terms used in the system are confusing and not very user-friendly. Users might not understand the jargon or technical terms.

Recommendations: We should use everyday language and follow familiar conventions for controls and buttons. This will make the system more relatable and easier to navigate for users.


3. User Control and Freedom

Issues: There’s no easy way to go back if a mistake is made. For instance, if a user accidentally deletes something, they have no option to undo that action.

Recommendations: Introducing an undo/redo option and clear cancel buttons would be really helpful. This gives users the freedom to correct their mistakes without stress.


4. Consistency and Standards

Issues: Similar buttons look alike, which can cause confusion. Users might not be sure about what each button does.

Recommendations: We should ensure that all buttons have a consistent look and clear labels. This helps users understand their functions at a glance.


5. Error Prevention

Issues: Users can easily make mistakes, especially since there are no prompts to confirm important actions. This could lead to accidental deletions or changes.

Recommendations: Adding checks to prevent errors and implementing confirmation dialogs for important actions would be beneficial. This way, users can think twice before committing to potentially harmful actions.


6. Recognition Rather Than Recall

Issues: Important information isn’t always visible, which forces users to remember details from different parts of the interface.

Recommendations: Keeping key information visible and adding tooltips for help would greatly assist users. Tooltips can provide additional context without overwhelming them.


7. Flexibility and Efficiency of Use

Issues: There are no options for users to customize their frequently used actions, which can slow down their workflow.

Recommendations: Allowing users to create shortcuts or personalize their interface would enhance their efficiency. This flexibility caters to individual preferences, making the system more user-friendly.


8. Aesthetic and Minimalist Design

Issues: The interface is cluttered with unnecessary information, which can distract users and reduce the visibility of essential content.

Recommendations: We should simplify the design by focusing on essential elements and removing distractions. A cleaner interface can improve overall usability.


9. Help Users Recognize, Diagnose, and Recover from Errors

Issues: Error messages are often hard to understand and use technical language, which can confuse users.

Recommendations: Rewriting error messages in simple terms and providing clear solutions would empower users to resolve issues quickly. This approach makes the system more approachable.


10. Help and Documentation

Issues: Finding help is difficult, and there’s no contextual assistance available when users encounter problems.

Recommendations: Improving the search feature for help and providing support right when users need it would enhance their experience. Contextual help can guide users without them having to search extensively.


By addressing these usability issues, we can create a more intuitive and user-friendly Employee Management System that enhances the overall user experience.
